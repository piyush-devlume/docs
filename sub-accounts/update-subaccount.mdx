---
title: "Update a sub-account's settings or status"
description: "Update a sub-account's name, email, description, rate limit, permissions, or enabled status. Only the fields you send are changed; others remain unchanged."
sidebarTitle: "Update sub-account"
---

Use this endpoint to update the configuration of an existing sub-account. Submit only the fields you want to change — all unspecified fields retain their current values. Setting `enabled` to `false` disables the sub-account, blocking all its API calls and trunk usage immediately.

**PUT** `https://api.vobiz.ai/api/v1/accounts/{auth_id}/sub-accounts/{sub_auth_id}`

## Path parameters

<ParamField path="auth_id" type="string" required>
  Your master account Auth ID (e.g., `MA_XXXXXX`).
</ParamField>

<ParamField path="sub_auth_id" type="string" required>
  The unique identifier of the sub-account to update (e.g., `SA_67401KW8`).
</ParamField>

## Body parameters

<ParamField body="name" type="string">
  Updated name for the sub-account.
</ParamField>

<ParamField body="email" type="string">
  Updated email address for the sub-account.
</ParamField>

<ParamField body="phone" type="string">
  Updated phone number for the sub-account.
</ParamField>

<ParamField body="description" type="string">
  Updated description of the sub-account purpose or usage.
</ParamField>

<ParamField body="rate_limit" type="integer">
  Updated API rate limit (requests per time period).
</ParamField>

<ParamField body="permissions" type="object">
  Updated permissions object. Contains boolean fields `calls` and `cdr`.
</ParamField>

<ParamField body="enabled" type="boolean">
  Set to `false` to disable the sub-account and block all its API calls.
</ParamField>

## Request examples

<CodeGroup>

```bash cURL
curl -X PUT 'https://api.vobiz.ai/api/v1/accounts/{auth_id}/sub-accounts/{sub_auth_id}' \
  -H 'X-Auth-ID: {auth_id}' \
  -H 'X-Auth-Token: {auth_token}' \
  -H 'Content-Type: application/json' \
  -d '{
    "description": "Updated permissions",
    "rate_limit": 750,
    "email": "updated@example.com",
    "enabled": true
  }'
```

```python Python
import requests

auth_id = "YOUR_AUTH_ID"
sub_auth_id = "SA_67401KW8"
url = f"https://api.vobiz.ai/api/v1/accounts/{auth_id}/sub-accounts/{sub_auth_id}"
headers = {
    "X-Auth-ID": auth_id,
    "X-Auth-Token": "YOUR_AUTH_TOKEN",
    "Content-Type": "application/json"
}
payload = {
    "description": "Updated permissions",
    "rate_limit": 750,
    "email": "updated@example.com",
    "enabled": True
}

response = requests.put(url, headers=headers, json=payload)
print(response.json())
```

```javascript Node.js
const axios = require("axios");

const authId = "YOUR_AUTH_ID";
const subAuthId = "SA_67401KW8";

const response = await axios.put(
  `https://api.vobiz.ai/api/v1/accounts/${authId}/sub-accounts/${subAuthId}`,
  {
    description: "Updated permissions",
    rate_limit: 750,
    email: "updated@example.com",
    enabled: true,
  },
  {
    headers: {
      "X-Auth-ID": authId,
      "X-Auth-Token": "YOUR_AUTH_TOKEN",
      "Content-Type": "application/json",
    },
  }
);

console.log(response.data);
```

</CodeGroup>

## Response

The response returns the full updated sub-account object, not just a success message. The `updated_at` timestamp reflects the modification time.

<ResponseField name="id" type="string">
  Unique identifier for the sub-account.
</ResponseField>

<ResponseField name="name" type="string">
  Updated or current name.
</ResponseField>

<ResponseField name="email" type="string">
  Updated or current email address.
</ResponseField>

<ResponseField name="description" type="string">
  Updated or current description.
</ResponseField>

<ResponseField name="permissions" type="object">
  Updated or current permissions.
</ResponseField>

<ResponseField name="rate_limit" type="integer">
  Updated or current API rate limit.
</ResponseField>

<ResponseField name="is_active" type="boolean">
  Current active status.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of the update. This value changes to reflect when you made this request.
</ResponseField>
