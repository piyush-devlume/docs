The Vobiz XML Builder is an interactive, browser-based tool that generates valid Vobiz XML without requiring you to write a single line of code. Select the elements you need, fill in the configuration fields, and the tool produces a ready-to-use XML document you can copy straight into your application or webhook response.

## What you can build

The XML Builder supports every major Vobiz XML element:

## How to use it

<Steps>
  <Step title="Open the XML Builder">
    Go to [docs.vobiz.ai/xml-builder](https://www.docs.vobiz.ai/xml-builder) in your browser.
  </Step>
  <Step title="Select the elements you need">
    Check the boxes for each XML element you want to include in your response. You can combine multiple elements — for example, `Speak` followed by `Gather` followed by `Hangup`.
  </Step>
  <Step title="Configure each element">
    Each selected element reveals a configuration panel with its available options — text to speak, voice, language, action URL, timeout, and so on. Fill in the fields relevant to your use case.
  </Step>
  <Step title="Copy the generated XML">
    The XML Builder updates the output pane in real time as you make changes. Click **Copy** to copy the complete XML document to your clipboard.
  </Step>
  <Step title="Paste into your application">
    Return the copied XML from your webhook handler. Set the HTTP `Content-Type` header to `application/xml` or `text/xml` so Vobiz parses it correctly.
  </Step>
</Steps>

## Example output

The following XML is what the builder produces for a `<Speak>` element with the default settings.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<Response>
    <Speak voice="WOMAN" language="en-US">
        Welcome to Vobiz.
    </Speak>
</Response>
```

Adjust the text, voice gender, and language in the builder, and the output updates immediately.

## When to use the XML Builder

<Tabs>
  <Tab title="Prototyping">
    Generate XML quickly to test a call flow idea without setting up a server or writing code. Paste the output directly into a static file served over HTTPS.
  </Tab>
  <Tab title="Learning the XML format">
    Use the builder to explore how each element and its attributes translate to valid XML. Comparing the live output with the reference docs is a fast way to learn the syntax.
  </Tab>
  <Tab title="Debugging">
    If a call flow is not behaving as expected, reproduce the intended XML in the builder to confirm the structure is correct, then compare it against what your application is returning.
  </Tab>
</Tabs>

<Tip>
  For production applications, generate XML programmatically in your server code based on business logic. The XML Builder is most useful for prototyping and learning — not as a replacement for a dynamic backend.
</Tip>
