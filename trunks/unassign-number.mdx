---
title: "Unassign a Phone Number from a SIP Trunk"
description: "Remove the link between a phone number and a SIP trunk. The number stays in your inventory but stops routing inbound calls through that trunk immediately."
sidebarTitle: "Unassign a number"
---

Use this endpoint to remove the assignment between a phone number and a SIP trunk. After unassignment, the number remains in your account inventory but no longer routes inbound calls through the trunk.

<Warning>
Unassignment takes effect immediately. Inbound calls to this number will fail until you reassign it to another trunk or configure an alternative routing destination.
</Warning>

## Request

```
DELETE https://api.vobiz.ai/api/v1/account/{auth_id}/numbers/{PHONE_NUMBER}/assign
```

**Headers**

| Header | Value |
|--------|-------|
| `X-Auth-ID` | Your account Auth ID |
| `X-Auth-Token` | Your account Auth Token |

No request body is required.

## Path parameters

<ParamField path="auth_id" type="string" required>
  Your Vobiz account identifier. Must match the `X-Auth-ID` header.
</ParamField>

<ParamField path="PHONE_NUMBER" type="string" required>
  The phone number to unassign, URL-encoded in E.164 format. Use `%2B` in place of `+` (e.g., `%2B1234567890`).
</ParamField>

<Note>
Always URL-encode the phone number. Replace `+` with `%2B`. Failing to do so will result in a `404` error.
</Note>

## Response

Returns `204 No Content` on success. No response body is returned.

| Status | Meaning |
|--------|---------|
| `204 No Content` | Number unassigned successfully. |
| `404 Not Found` | Phone number not found or not assigned to any trunk. |
| `400 Bad Request` | Phone number format is invalid or not URL-encoded correctly. |

## Examples

<CodeGroup>

```bash cURL
curl -X DELETE \
  "https://api.vobiz.ai/api/v1/account/{auth_id}/numbers/%2B1234567890/assign" \
  -H "X-Auth-ID: {auth_id}" \
  -H "X-Auth-Token: {AUTH_TOKEN}"
```

```python Python
import requests
from urllib.parse import quote

phone_number = "+1234567890"
encoded_number = quote(phone_number, safe="")

url = f"https://api.vobiz.ai/api/v1/account/{{auth_id}}/numbers/{encoded_number}/assign"
headers = {
    "X-Auth-ID": "{auth_id}",
    "X-Auth-Token": "{AUTH_TOKEN}"
}

response = requests.delete(url, headers=headers)
if response.status_code == 204:
    print("Number unassigned successfully.")
```

```javascript Node.js
const phoneNumber = "+1234567890";
const encodedNumber = encodeURIComponent(phoneNumber);

const response = await fetch(
  `https://api.vobiz.ai/api/v1/account/{auth_id}/numbers/${encodedNumber}/assign`,
  {
    method: "DELETE",
    headers: {
      "X-Auth-ID": "{auth_id}",
      "X-Auth-Token": "{AUTH_TOKEN}"
    }
  }
);

if (response.status === 204) {
  console.log("Number unassigned successfully.");
}
```

</CodeGroup>

**Error responses**

```json
// 404 — number not found or not assigned
{
  "error": "Number not found",
  "message": "The phone number could not be found or is not assigned to any trunk."
}
```

```json
// 400 — bad number format
{
  "error": "invalid request",
  "message": "The phone number format is invalid. Ensure it is URL-encoded correctly."
}
```

<Note>
Unassigning a number stops routing but does not release it from your account. Number rental charges continue until you release the number. Reassign it to a different trunk using the [Assign Number](/trunks/assign-number) endpoint if you want to continue receiving calls.
</Note>
