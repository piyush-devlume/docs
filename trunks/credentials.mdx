Credentials provide username/password-based SIP digest authentication for your trunks. Each credential consists of a username and password that SIP clients present when registering or placing calls. You can create multiple credentials per account, allowing different devices, users, or applications to connect with independent authentication details.

<Note>
Passwords are **write-only**. The API never returns a password in any response. If you lose a credential's password, you must update it — you cannot retrieve it.
</Note>

## The credential object

Each credential has the following fields:

<ResponseField name="id" type="string (UUID)">
  Unique identifier for the credential. Auto-generated on creation.
</ResponseField>

<ResponseField name="trunk_id" type="string (UUID)">
  The trunk this credential is associated with.
</ResponseField>

<ResponseField name="username" type="string">
  SIP username for authentication. Unique within the account. **Cannot be changed after creation.**
</ResponseField>

<ResponseField name="enabled" type="boolean">
  Whether the credential is active. Default: `true`.
</ResponseField>

<ResponseField name="description" type="string">
  Optional description identifying the device or user this credential belongs to.
</ResponseField>

<ResponseField name="created_at" type="string (ISO 8601)">
  UTC creation timestamp.
</ResponseField>

<ResponseField name="updated_at" type="string (ISO 8601)">
  UTC last-updated timestamp.
</ResponseField>

**Example credential object**

```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "trunk_id": "bfab10fb-cb97-488b-9c63-989c32980b0f",
  "username": "trunk_user_001",
  "enabled": true,
  "description": "Primary trunk credential for production",
  "created_at": "2025-01-15T10:35:20Z",
  "updated_at": "2025-01-15T10:35:20Z"
}
```

## Create a credential

```
POST https://api.vobiz.ai/api/v1/account/{auth_id}/credentials
```

**Headers:** `X-Auth-ID`, `X-Auth-Token`, `Content-Type: application/json`

### Parameters

<ParamField body="username" type="string" required>
  SIP username. Must be unique within the account. Use descriptive names like `office_phone_01`. Cannot be changed after creation.
</ParamField>

<ParamField body="password" type="string" required>
  SIP password. Minimum 8 characters. Use 12+ characters with mixed case, numbers, and symbols.
</ParamField>

<ParamField body="enabled" type="boolean">
  Whether the credential is active immediately. Default: `true`.
</ParamField>

<ParamField body="description" type="string">
  Optional description (e.g., `"Main office desk phone"`).
</ParamField>

<CodeGroup>

```bash cURL
curl -X POST https://api.vobiz.ai/api/v1/account/{Auth_ID}/credentials \
  -H "X-Auth-ID: YOUR_AUTH_ID" \
  -H "X-Auth-Token: YOUR_AUTH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "trunk_user_001",
    "password": "Vbz!2025$SecureP@ss",
    "enabled": true,
    "description": "Primary credential"
  }'
```

```python Python
import requests

url = "https://api.vobiz.ai/api/v1/account/{Auth_ID}/credentials"
headers = {
    "X-Auth-ID": "YOUR_AUTH_ID",
    "X-Auth-Token": "YOUR_AUTH_TOKEN",
    "Content-Type": "application/json"
}
payload = {
    "username": "trunk_user_001",
    "password": "Vbz!2025$SecureP@ss",
    "enabled": True,
    "description": "Primary credential"
}

response = requests.post(url, json=payload, headers=headers)
credential = response.json()
print(credential["id"])
```

```javascript Node.js
const response = await fetch(
  "https://api.vobiz.ai/api/v1/account/{Auth_ID}/credentials",
  {
    method: "POST",
    headers: {
      "X-Auth-ID": "YOUR_AUTH_ID",
      "X-Auth-Token": "YOUR_AUTH_TOKEN",
      "Content-Type": "application/json"
    },
    body: JSON.stringify({
      username: "trunk_user_001",
      password: "Vbz!2025$SecureP@ss",
      enabled: true,
      description: "Primary credential"
    })
  }
);

const credential = await response.json();
console.log(credential.id);
```

</CodeGroup>

**Response (201 Created)**

```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "username": "trunk_user_001",
  "enabled": true,
  "description": "Primary credential",
  "created_at": "2025-01-22T10:30:00Z",
  "updated_at": "2025-01-22T10:30:00Z"
}
```

<Warning>
Save the password securely immediately after creation. It will never appear in any subsequent API response.
</Warning>

## List all credentials

Retrieve a paginated list of all credentials in the account.

```
GET https://api.vobiz.ai/api/v1/account/{auth_id}/trunks/credentials
```

**Query parameters**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `limit` | integer | `20` | Results per page. |
| `offset` | integer | `0` | Results to skip for pagination. |

<CodeGroup>

```bash cURL
curl -X GET "https://api.vobiz.ai/api/v1/account/{Auth_ID}/trunks/credentials?limit=20&offset=0" \
  -H "X-Auth-ID: YOUR_AUTH_ID" \
  -H "X-Auth-Token: YOUR_AUTH_TOKEN"
```

```python Python
import requests

url = "https://api.vobiz.ai/api/v1/account/{Auth_ID}/trunks/credentials"
headers = {
    "X-Auth-ID": "YOUR_AUTH_ID",
    "X-Auth-Token": "YOUR_AUTH_TOKEN"
}
params = {"limit": 20, "offset": 0}

response = requests.get(url, headers=headers, params=params)
data = response.json()

for cred in data["objects"]:
    print(cred["id"], cred["username"], cred["enabled"])
```

```javascript Node.js
const response = await fetch(
  "https://api.vobiz.ai/api/v1/account/{Auth_ID}/trunks/credentials?limit=20&offset=0",
  {
    headers: {
      "X-Auth-ID": "YOUR_AUTH_ID",
      "X-Auth-Token": "YOUR_AUTH_TOKEN"
    }
  }
);

const data = await response.json();
data.objects.forEach(c => console.log(c.id, c.username, c.enabled));
```

## Update a credential

Rotate a password, change the enabled state, or update the description. The `username` field is immutable.

```
PUT https://api.vobiz.ai/api/v1/account/{auth_id}/credentials/{credential_id}
```

### Parameters

<ParamField body="password" type="string">
  New SIP password. Minimum 8 characters.
</ParamField>

<ParamField body="enabled" type="boolean">
  `false` to disable, `true` to re-enable.
</ParamField>

<ParamField body="description" type="string">
  Updated description.
</ParamField>

<CodeGroup>

```bash cURL — rotate password
curl -X PUT \
  https://api.vobiz.ai/api/v1/account/{Auth_ID}/credentials/a1b2c3d4-e5f6-7890-abcd-ef1234567890 \
  -H "X-Auth-ID: YOUR_AUTH_ID" \
  -H "X-Auth-Token: YOUR_AUTH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "password": "UpdatedSecurePassword456!",
    "description": "Password rotated 2025-01-22"
  }'
```

```bash cURL — disable credential
curl -X PUT \
  https://api.vobiz.ai/api/v1/account/{Auth_ID}/credentials/a1b2c3d4-e5f6-7890-abcd-ef1234567890 \
  -H "X-Auth-ID: YOUR_AUTH_ID" \
  -H "X-Auth-Token: YOUR_AUTH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"enabled": false}'
```

```python Python
import requests

cred_id = "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
url = f"https://api.vobiz.ai/api/v1/account/{{Auth_ID}}/credentials/{cred_id}"
headers = {
    "X-Auth-ID": "YOUR_AUTH_ID",
    "X-Auth-Token": "YOUR_AUTH_TOKEN",
    "Content-Type": "application/json"
}
payload = {
    "password": "UpdatedSecurePassword456!",
    "description": "Password rotated 2025-01-22"
}

response = requests.put(url, json=payload, headers=headers)
print(response.json()["updated_at"])
```

```javascript Node.js
const credId = "a1b2c3d4-e5f6-7890-abcd-ef1234567890";
const response = await fetch(
  `https://api.vobiz.ai/api/v1/account/{Auth_ID}/credentials/${credId}`,
  {
    method: "PUT",
    headers: {
      "X-Auth-ID": "YOUR_AUTH_ID",
      "X-Auth-Token": "YOUR_AUTH_TOKEN",
      "Content-Type": "application/json"
    },
    body: JSON.stringify({ enabled: false })
  }
);

const cred = await response.json();
console.log(cred.enabled, cred.updated_at);
```

</CodeGroup>

**Response (200 OK)**

```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "username": "trunk_user_001",
  "enabled": false,
  "description": "Temporarily disabled for maintenance",
  "created_at": "2025-01-15T10:35:20Z",
  "updated_at": "2025-01-22T11:15:45Z"
}
```

## Delete a credential

Permanently remove a credential. This immediately blocks authentication using the associated username and password.

```
DELETE https://api.vobiz.ai/api/v1/account/{auth_id}/credentials/{credential_id}
```

<Warning>
Deleting a credential immediately prevents authentication for any device using it. Active calls using this credential may be disconnected. This action cannot be undone. Disable the credential instead if you might need it again.
</Warning>

Returns `204 No Content` on success.

<CodeGroup>

```bash cURL
curl -X DELETE \
  https://api.vobiz.ai/api/v1/account/{Auth_ID}/credentials/a1b2c3d4-e5f6-7890-abcd-ef1234567890 \
  -H "X-Auth-ID: YOUR_AUTH_ID" \
  -H "X-Auth-Token: YOUR_AUTH_TOKEN"
```

```python Python
import requests

cred_id = "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
url = f"https://api.vobiz.ai/api/v1/account/{{Auth_ID}}/credentials/{cred_id}"
headers = {
    "X-Auth-ID": "YOUR_AUTH_ID",
    "X-Auth-Token": "YOUR_AUTH_TOKEN"
}

response = requests.delete(url, headers=headers)
if response.status_code == 204:
    print("Credential deleted.")
```

```javascript Node.js
const credId = "a1b2c3d4-e5f6-7890-abcd-ef1234567890";
const response = await fetch(
  `https://api.vobiz.ai/api/v1/account/{Auth_ID}/credentials/${credId}`,
  {
    method: "DELETE",
    headers: {
      "X-Auth-ID": "YOUR_AUTH_ID",
      "X-Auth-Token": "YOUR_AUTH_TOKEN"
    }
  }
);

if (response.status === 204) {
  console.log("Credential deleted.");
}
```

</CodeGroup>

| Status | Meaning |
|--------|---------|
| `204 No Content` | Credential deleted. |
| `404 Not Found` | Credential does not exist. |
| `409 Conflict` | Credential is in use by active calls. |

## Best practices

- Rotate passwords quarterly or whenever a device is compromised.
- Disable (`enabled: false`) instead of deleting to preserve an audit trail.
- Use descriptive usernames that identify the device or user (e.g., `sales_desk_nyc`).
- Combine credentials with [IP ACLs](/trunks/ip-acl) for layered authentication security.
