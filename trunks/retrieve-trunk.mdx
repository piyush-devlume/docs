Use this endpoint to fetch the complete configuration of a single trunk by its `trunk_id`. This is useful for verifying settings, checking rate limits, or confirming whether a trunk is currently enabled.

## Request

```
GET https://api.vobiz.ai/api/v1/account/{auth_id}/trunks/{trunk_id}
```

**Headers**

| Header | Value |
|--------|-------|
| `X-Auth-ID` | Your account Auth ID |
| `X-Auth-Token` | Your account Auth Token |

## Path parameters

<ParamField path="auth_id" type="string" required>
  Your Vobiz account identifier.
</ParamField>

<ParamField path="trunk_id" type="string" required>
  The UUID of the trunk you want to retrieve.
</ParamField>

## Response

Returns the complete trunk object on success.

<ResponseField name="trunk_id" type="string (UUID)">
  Unique identifier for the trunk.
</ResponseField>

<ResponseField name="auth_id" type="string">
  Account identifier that owns this trunk.
</ResponseField>

<ResponseField name="name" type="string">
  Descriptive name for the trunk.
</ResponseField>

<ResponseField name="description" type="string">
  Optional description.
</ResponseField>

<ResponseField name="trunk_domain" type="string">
  Auto-generated SIP domain for this trunk.
</ResponseField>

<ResponseField name="trunk_status" type="string">
  Current operational status of the trunk.
</ResponseField>

<ResponseField name="trunk_direction" type="string">
  Call direction supported: `outbound`, `inbound`, or `both`.
</ResponseField>

<ResponseField name="secure" type="boolean">
  Whether TLS/SRTP is enabled.
</ResponseField>

<ResponseField name="transport" type="string">
  SIP transport protocol in use.
</ResponseField>

<ResponseField name="concurrent_calls_limit" type="integer">
  Maximum simultaneous calls.
</ResponseField>

<ResponseField name="cps_limit" type="integer">
  Calls-per-second rate limit.
</ResponseField>

<ResponseField name="ipacl_uuid" type="string (UUID)">
  Associated IP ACL UUID, if configured.
</ResponseField>

<ResponseField name="credential_uuid" type="string (UUID)">
  Associated credential UUID, if configured.
</ResponseField>

<ResponseField name="primary_uri_uuid" type="string (UUID)">
  Primary origination URI UUID.
</ResponseField>

<ResponseField name="fallback_uri_uuid" type="string (UUID)">
  Fallback origination URI UUID.
</ResponseField>

<ResponseField name="inbound_destination" type="string">
  SIP URI for inbound call forwarding.
</ResponseField>

<ResponseField name="created_at" type="string (ISO 8601)">
  UTC creation timestamp.
</ResponseField>

<ResponseField name="updated_at" type="string (ISO 8601)">
  UTC last-updated timestamp.
</ResponseField>

## Examples

<CodeGroup>

```bash cURL
curl -X GET \
  https://api.vobiz.ai/api/v1/account/{Auth_ID}/trunks/bfab10fb-cb97-488b-9c63-989c32980b0f \
  -H "X-Auth-ID: YOUR_AUTH_ID" \
  -H "X-Auth-Token: YOUR_AUTH_TOKEN"
```

```python Python
import requests

trunk_id = "bfab10fb-cb97-488b-9c63-989c32980b0f"
url = f"https://api.vobiz.ai/api/v1/account/{{Auth_ID}}/trunks/{trunk_id}"
headers = {
    "X-Auth-ID": "YOUR_AUTH_ID",
    "X-Auth-Token": "YOUR_AUTH_TOKEN"
}

response = requests.get(url, headers=headers)
trunk = response.json()
print(trunk["name"], trunk["trunk_status"])
```

```javascript Node.js
const trunkId = "bfab10fb-cb97-488b-9c63-989c32980b0f";
const response = await fetch(
  `https://api.vobiz.ai/api/v1/account/{Auth_ID}/trunks/${trunkId}`,
  {
    headers: {
      "X-Auth-ID": "YOUR_AUTH_ID",
      "X-Auth-Token": "YOUR_AUTH_TOKEN"
    }
  }
);

const trunk = await response.json();
console.log(trunk.name, trunk.trunk_status);
```

</CodeGroup>
