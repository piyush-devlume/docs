IP Access Control Lists (IP ACLs) allow you to whitelist specific IPv4 addresses so that calls originating from those addresses authenticate to your trunk without requiring a username or password. This is ideal for PBX systems, SIP gateways, and carrier connections that operate from static public IP addresses.

<Note>
IP ACLs only support IPv4 addresses. IPv6 is not currently available. For devices behind NAT, whitelist the router's public IP, not the device's internal IP.
</Note>

## When to use IP ACLs vs credentials

| | IP ACLs | Credentials |
|---|---|---|
| Best for | Static IPs (PBX, SBCs, carriers) | Dynamic IPs (remote workers, mobile) |
| Setup | Simple — no client config needed | Requires configuring username/password |
| Security | Good (IP-based) | Strong (cryptographic digest) |
| NAT compatibility | Limited — public IP only | Full — works from any network |

<Tip>
For maximum security, use both IP ACLs and credentials together. A caller must originate from a whitelisted IP **and** supply valid credentials.
</Tip>

## The IP ACL object

<ResponseField name="id" type="string (UUID)">
  Unique identifier for the IP ACL entry. Auto-generated on creation.
</ResponseField>

<ResponseField name="trunk_id" type="string (UUID)">
  The trunk this IP ACL entry is associated with.
</ResponseField>

<ResponseField name="ip_address" type="string">
  Whitelisted IPv4 address in dotted-decimal notation (e.g., `203.0.113.50`).
</ResponseField>

<ResponseField name="description" type="string">
  Optional description (e.g., `"Office static IP"` or `"PBX server"`).
</ResponseField>

<ResponseField name="enabled" type="boolean">
  Whether the entry is active. Default: `true`.
</ResponseField>

<ResponseField name="created_at" type="string (ISO 8601)">
  UTC creation timestamp.
</ResponseField>

<ResponseField name="updated_at" type="string (ISO 8601)">
  UTC last-updated timestamp.
</ResponseField>

**Example IP ACL object**

```json
{
  "id": "c1d2e3f4-a5b6-7890-cdef-1234567890ab",
  "trunk_id": "bfab10fb-cb97-488b-9c63-989c32980b0f",
  "ip_address": "203.0.113.50",
  "description": "Office static IP",
  "enabled": true,
  "created_at": "2025-01-15T10:40:15Z",
  "updated_at": "2025-01-15T10:40:15Z"
}
```

## Create an IP ACL entry

```
POST https://api.vobiz.ai/api/v1/account/{auth_id}/ip-acl
```

**Headers:** `X-Auth-ID`, `X-Auth-Token`, `Content-Type: application/json`

### Parameters

<ParamField body="ip_address" type="string" required>
  IPv4 address in dotted-decimal notation (e.g., `203.0.113.50`). Must be a valid IPv4 format.
</ParamField>

<ParamField body="enabled" type="boolean">
  Whether the entry is active immediately. Default: `true`.
</ParamField>

<ParamField body="description" type="string">
  Optional label identifying the IP's purpose (e.g., `"FreePBX server in data center"`).
</ParamField>

<CodeGroup>

```bash cURL
curl -X POST https://api.vobiz.ai/api/v1/account/{Auth_ID}/ip-acl \
  -H "X-Auth-ID: YOUR_AUTH_ID" \
  -H "X-Auth-Token: YOUR_AUTH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "ip_address": "203.0.113.50",
    "description": "Office static IP",
    "enabled": true
  }'
```

```python Python
import requests

url = "https://api.vobiz.ai/api/v1/account/{Auth_ID}/ip-acl"
headers = {
    "X-Auth-ID": "YOUR_AUTH_ID",
    "X-Auth-Token": "YOUR_AUTH_TOKEN",
    "Content-Type": "application/json"
}
payload = {
    "ip_address": "203.0.113.50",
    "description": "Office static IP",
    "enabled": True
}

response = requests.post(url, json=payload, headers=headers)
acl = response.json()
print(acl["id"])
```

```javascript Node.js
const response = await fetch(
  "https://api.vobiz.ai/api/v1/account/{Auth_ID}/ip-acl",
  {
    method: "POST",
    headers: {
      "X-Auth-ID": "YOUR_AUTH_ID",
      "X-Auth-Token": "YOUR_AUTH_TOKEN",
      "Content-Type": "application/json"
    },
    body: JSON.stringify({
      ip_address: "203.0.113.50",
      description: "Office static IP",
      enabled: true
    })
  }
);

const acl = await response.json();
console.log(acl.id);
```

</CodeGroup>

**Response (201 Created)**

```json
{
  "id": "c1d2e3f4-a5b6-7890-cdef-1234567890ab",
  "ip_address": "203.0.113.50",
  "description": "Office static IP",
  "enabled": true,
  "created_at": "2025-01-22T11:20:30Z",
  "updated_at": "2025-01-22T11:20:30Z"
}
```

<Tip>
If you're unsure of your public IP address, run `curl https://api.ipify.org` from the device you want to whitelist.
</Tip>

## List all IP ACL entries

```
GET https://api.vobiz.ai/api/v1/account/{auth_id}/trunks/ip-acl
```

**Query parameters**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `limit` | integer | `20` | Results per page. Maximum: `100`. |
| `offset` | integer | `0` | Results to skip for pagination. |

<CodeGroup>

```bash cURL
curl -X GET "https://api.vobiz.ai/api/v1/account/{Auth_ID}/trunks/ip-acl?limit=20&offset=0" \
  -H "X-Auth-ID: YOUR_AUTH_ID" \
  -H "X-Auth-Token: YOUR_AUTH_TOKEN"
```

```python Python
import requests

url = "https://api.vobiz.ai/api/v1/account/{Auth_ID}/trunks/ip-acl"
headers = {
    "X-Auth-ID": "YOUR_AUTH_ID",
    "X-Auth-Token": "YOUR_AUTH_TOKEN"
}
params = {"limit": 20, "offset": 0}

response = requests.get(url, headers=headers, params=params)
data = response.json()

for acl in data["objects"]:
    print(acl["id"], acl["ip_address"], acl["enabled"])
```

## Update an IP ACL entry

Change the whitelisted IP address, update the description, or enable/disable the entry.

```
PUT https://api.vobiz.ai/api/v1/account/{auth_id}/ip-acl/{ip_acl_id}
```

### Parameters

All parameters are optional. Include only the fields you want to change.

<ParamField body="ip_address" type="string">
  New IPv4 address in dotted-decimal notation.
</ParamField>

<ParamField body="enabled" type="boolean">
  `false` to disable, `true` to re-enable.
</ParamField>

<ParamField body="description" type="string">
  Updated description.
</ParamField>

<CodeGroup>

```bash cURL — update IP address
curl -X PUT \
  https://api.vobiz.ai/api/v1/account/{Auth_ID}/ip-acl/c1d2e3f4-a5b6-7890-cdef-1234567890ab \
  -H "X-Auth-ID: YOUR_AUTH_ID" \
  -H "X-Auth-Token: YOUR_AUTH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "ip_address": "192.168.1.200",
    "description": "Updated office IP after ISP change"
  }'
```

```bash cURL — disable entry
curl -X PUT \
  https://api.vobiz.ai/api/v1/account/{Auth_ID}/ip-acl/c1d2e3f4-a5b6-7890-cdef-1234567890ab \
  -H "X-Auth-ID: YOUR_AUTH_ID" \
  -H "X-Auth-Token: YOUR_AUTH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"enabled": false}'
```

```python Python
import requests

acl_id = "c1d2e3f4-a5b6-7890-cdef-1234567890ab"
url = f"https://api.vobiz.ai/api/v1/account/{{Auth_ID}}/ip-acl/{acl_id}"
headers = {
    "X-Auth-ID": "YOUR_AUTH_ID",
    "X-Auth-Token": "YOUR_AUTH_TOKEN",
    "Content-Type": "application/json"
}
payload = {
    "ip_address": "192.168.1.200",
    "description": "Updated office IP after ISP change"
}

response = requests.put(url, json=payload, headers=headers)
print(response.json()["updated_at"])
```

```javascript Node.js
const aclId = "c1d2e3f4-a5b6-7890-cdef-1234567890ab";
const response = await fetch(
  `https://api.vobiz.ai/api/v1/account/{Auth_ID}/ip-acl/${aclId}`,
  {
    method: "PUT",
    headers: {
      "X-Auth-ID": "YOUR_AUTH_ID",
      "X-Auth-Token": "YOUR_AUTH_TOKEN",
      "Content-Type": "application/json"
    },
    body: JSON.stringify({ enabled: false })
  }
);

const acl = await response.json();
console.log(acl.enabled, acl.updated_at);
```

</CodeGroup>

**Response (200 OK)**

```json
{
  "id": "c1d2e3f4-a5b6-7890-cdef-1234567890ab",
  "ip_address": "192.168.1.200",
  "description": "Updated office IP after ISP change",
  "enabled": true,
  "created_at": "2025-01-15T10:40:15Z",
  "updated_at": "2025-01-22T11:45:20Z"
}
```

<Warning>
When changing an IP address, existing authenticated connections may be disconnected. Add the new IP first, verify connectivity, then disable or delete the old entry to avoid a service gap.
</Warning>

## Delete an IP ACL entry

Permanently remove an IP address from the whitelist.

```
DELETE https://api.vobiz.ai/api/v1/account/{auth_id}/ip-acl/{ip_acl_id}
```

<Warning>
Deletion is permanent. Calls from the removed IP address will be rejected immediately unless another authentication method is configured. Consider disabling the entry instead.
</Warning>

Returns `204 No Content` on success.

<CodeGroup>

```bash cURL
curl -X DELETE \
  https://api.vobiz.ai/api/v1/account/{Auth_ID}/ip-acl/c1d2e3f4-a5b6-7890-cdef-1234567890ab \
  -H "X-Auth-ID: YOUR_AUTH_ID" \
  -H "X-Auth-Token: YOUR_AUTH_TOKEN"
```

```python Python
import requests

acl_id = "c1d2e3f4-a5b6-7890-cdef-1234567890ab"
url = f"https://api.vobiz.ai/api/v1/account/{{Auth_ID}}/ip-acl/{acl_id}"
headers = {
    "X-Auth-ID": "YOUR_AUTH_ID",
    "X-Auth-Token": "YOUR_AUTH_TOKEN"
}

response = requests.delete(url, headers=headers)
if response.status_code == 204:
    print("IP ACL entry deleted.")
```

```javascript Node.js
const aclId = "c1d2e3f4-a5b6-7890-cdef-1234567890ab";
const response = await fetch(
  `https://api.vobiz.ai/api/v1/account/{Auth_ID}/ip-acl/${aclId}`,
  {
    method: "DELETE",
    headers: {
      "X-Auth-ID": "YOUR_AUTH_ID",
      "X-Auth-Token": "YOUR_AUTH_TOKEN"
    }
  }
);

if (response.status === 204) {
  console.log("IP ACL entry deleted.");
}
```

</CodeGroup>

| Status | Meaning |
|--------|---------|
| `204 No Content` | Entry deleted. |
| `404 Not Found` | Entry does not exist or was already deleted. |

## Best practices

- Only whitelist IP addresses you own and control.
- Add descriptive labels to every entry to make security audits easier.
- Review your IP ACL list regularly and remove entries that are no longer needed.
- Disable rather than delete when temporarily blocking access to preserve the configuration.
- Plan IP address changes during maintenance windows to avoid service disruption.
