---
title: "Update an existing Vobiz application's configuration"
description: "Modify a subset of fields on an existing application. Unspecified fields remain unchanged. Note that sub_account cannot be changed after creation."
sidebarTitle: "Update application"
---

Use this endpoint to update one or more fields on an existing application. You only need to include the fields you want to change — all other fields remain at their current values. Note that the `sub_account` field cannot be changed after an application is created.

**POST** `https://api.vobiz.ai/api/v1/Account/{authID}/Application/{appID}/`

<Note>
  The `sub_account` field cannot be changed after an application is created. All other fields support partial updates.
</Note>

## Path parameters

<ParamField path="authID" type="string" required>
  Your account Auth ID (e.g., `MA_XXXXXX`).
</ParamField>

<ParamField path="appID" type="string" required>
  The unique identifier of the application to update (e.g., `12345678`).
</ParamField>

## Body parameters

<ParamField body="answer_url" type="string">
  The URL invoked when a call executes this application.
</ParamField>

<ParamField body="answer_method" type="string">
  HTTP method for `answer_url`. Default: `POST`.
</ParamField>

<ParamField body="hangup_url" type="string">
  URL notified when the call hangs up.
</ParamField>

<ParamField body="hangup_method" type="string">
  HTTP method for `hangup_url`. Default: `POST`.
</ParamField>

<ParamField body="fallback_answer_url" type="string">
  Invoked by Vobiz only if `answer_url` is unavailable or returns invalid XML.
</ParamField>

<ParamField body="fallback_method" type="string">
  HTTP method for `fallback_answer_url`. Default: `POST`.
</ParamField>

<ParamField body="message_url" type="string">
  URL notified when an inbound message is received.
</ParamField>

<ParamField body="message_method" type="string">
  HTTP method for `message_url`. Default: `POST`.
</ParamField>

<ParamField body="default_number_app" type="boolean">
  When `true`, newly created phone numbers without an explicit `app_id` are associated with this application.
</ParamField>

<ParamField body="default_endpoint_app" type="boolean">
  When `true`, newly created SIP endpoints without an explicit `app_id` are associated with this application.
</ParamField>

<ParamField body="application_type" type="string">
  The type of application (e.g., `voice`, `messaging`).
</ParamField>

<ParamField body="default_app" type="boolean">
  Whether this is the default application for the account.
</ParamField>

<ParamField body="enabled" type="boolean">
  Whether the application is enabled.
</ParamField>

<ParamField body="log_incoming_messages" type="boolean">
  Controls whether incoming messages to numbers associated with this application are logged.
</ParamField>

<ParamField body="sip_transfer_url" type="string">
  URL notified during a SIP transfer.
</ParamField>

<ParamField body="sip_transfer_method" type="string">
  HTTP method for SIP transfer callbacks.
</ParamField>

## Request examples

<CodeGroup>

```bash cURL
curl -X POST https://api.vobiz.ai/api/v1/Account/{Auth_ID}/Application/12345678/ \
  -H "X-Auth-ID: YOUR_AUTH_ID" \
  -H "X-Auth-Token: YOUR_AUTH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "app_name": "Updated IVR System",
    "default_number_app": true
  }'
```

```python Python
import requests

auth_id = "YOUR_AUTH_ID"
app_id = "12345678"
url = f"https://api.vobiz.ai/api/v1/Account/{auth_id}/Application/{app_id}/"
headers = {
    "X-Auth-ID": auth_id,
    "X-Auth-Token": "YOUR_AUTH_TOKEN",
    "Content-Type": "application/json"
}
payload = {
    "app_name": "Updated IVR System",
    "default_number_app": True
}

response = requests.post(url, headers=headers, json=payload)
print(response.json())
```

```javascript Node.js
const axios = require("axios");

const authId = "YOUR_AUTH_ID";
const appId = "12345678";

const response = await axios.post(
  `https://api.vobiz.ai/api/v1/Account/${authId}/Application/${appId}/`,
  {
    app_name: "Updated IVR System",
    default_number_app: true,
  },
  {
    headers: {
      "X-Auth-ID": authId,
      "X-Auth-Token": "YOUR_AUTH_TOKEN",
      "Content-Type": "application/json",
    },
  }
);

console.log(response.data);
```

</CodeGroup>
