Use this endpoint to retrieve all configuration details for a specific application. Pass the `app_id` in the URL path to get the full application object, including webhook URLs, HTTP methods, SIP settings, and default routing flags. No additional parameters are required.

**GET** `https://api.vobiz.ai/api/v1/Account/{authID}/Application/{appID}/`

## Path parameters

<ParamField path="authID" type="string" required>
  Your account Auth ID (e.g., `MA_XXXXXX`).
</ParamField>

<ParamField path="appID" type="string" required>
  The unique identifier of the application to retrieve (e.g., `12345678`).
</ParamField>

## Request examples

<CodeGroup>

```bash cURL
curl -X GET https://api.vobiz.ai/api/v1/Account/{Auth_ID}/Application/12345678/ \
  -H "X-Auth-ID: YOUR_AUTH_ID" \
  -H "X-Auth-Token: YOUR_AUTH_TOKEN"
```

```python Python
import requests

auth_id = "YOUR_AUTH_ID"
app_id = "12345678"
url = f"https://api.vobiz.ai/api/v1/Account/{auth_id}/Application/{app_id}/"
headers = {
    "X-Auth-ID": auth_id,
    "X-Auth-Token": "YOUR_AUTH_TOKEN"
}

response = requests.get(url, headers=headers)
print(response.json())
```

```javascript Node.js
const axios = require("axios");

const authId = "YOUR_AUTH_ID";
const appId = "12345678";

const response = await axios.get(
  `https://api.vobiz.ai/api/v1/Account/${authId}/Application/${appId}/`,
  {
    headers: {
      "X-Auth-ID": authId,
      "X-Auth-Token": "YOUR_AUTH_TOKEN",
    },
  }
);

console.log(response.data);
```

</CodeGroup>

## Response fields

<ResponseField name="app_id" type="string">
  Unique identifier for the application.
</ResponseField>

<ResponseField name="app_name" type="string">
  Name of the application.
</ResponseField>

<ResponseField name="application_type" type="string">
  The type of application (e.g., `voice`, `messaging`).
</ResponseField>

<ResponseField name="enabled" type="boolean">
  Whether the application is currently enabled.
</ResponseField>

<ResponseField name="answer_url" type="string">
  URL Vobiz requests when a call is answered.
</ResponseField>

<ResponseField name="answer_method" type="string">
  HTTP method for `answer_url`.
</ResponseField>

<ResponseField name="hangup_url" type="string">
  URL notified when a call ends.
</ResponseField>

<ResponseField name="hangup_method" type="string">
  HTTP method for `hangup_url`.
</ResponseField>

<ResponseField name="fallback_answer_url" type="string">
  Fallback URL used when `answer_url` is unavailable.
</ResponseField>

<ResponseField name="fallback_method" type="string">
  HTTP method for `fallback_answer_url`.
</ResponseField>

<ResponseField name="message_url" type="string">
  URL notified when an inbound message is received.
</ResponseField>

<ResponseField name="message_method" type="string">
  HTTP method for `message_url`.
</ResponseField>

<ResponseField name="sip_uri" type="string">
  SIP URI for the application.
</ResponseField>

<ResponseField name="default_number_app" type="boolean">
  Whether this application is the default for new phone numbers.
</ResponseField>

<ResponseField name="default_endpoint_app" type="boolean">
  Whether this application is the default for new SIP endpoints.
</ResponseField>

<ResponseField name="resource_uri" type="string">
  API resource URI for this application.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of creation.
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of last update.
</ResponseField>
