This example walks you through building a multi-level interactive voice response (IVR) menu for a company main line. Callers choose from Sales, Technical Support, Billing, or Account Management using keypad input. Transfer numbers are updated at runtime via a REST API—no server restart required—and full call analytics track which menu options callers select most often.

<Info>
The full source code is available at [github.com/vobiz-ai/Vobiz-IVR-XML-Python](https://github.com/vobiz-ai/Vobiz-IVR-XML-Python).
</Info>

## Quick start

```bash
git clone https://github.com/vobiz-ai/Vobiz-IVR-XML-Python.git
cd Vobiz-IVR-XML-Python
cp .env.example .env
pip install -r requirements.txt
python server.py
```

After the server starts, set the **Answer URL** on your Vobiz application to `https://<your-public-url>/answer` using POST method.

## VobizXML verbs used

This example uses the following XML verbs: `<Gather inputType="dtmf">`, `<Speak>`, `<Redirect>`, `<Dial><Number>`, and `<Hangup>`.

## Call flow

```text
/answer  →  Main menu
              1 → Sales
              │     1 → Products & pricing (spoken)
              │     2 → Request a demo → Hangup
              │     3 → Connect to sales rep → <Dial>
              │     9 → Back to main menu
              2 → Technical Support
              │     1 → API issues → raise ticket option
              │     2 → Call quality → <Dial>
              │     3 → Account access (spoken)
              │     9 → Back
              3 → Billing
              │     1 → Balance info (spoken)
              │     2 → Payment issues → <Dial>
              │     3 → Invoice / GST info (spoken)
              │     9 → Back
              4 → Account Management
              │     1 → Upgrade plan → <Dial>
              │     2 → Update details (spoken)
              │     3 → Cancel → double confirm
              │     9 → Back
              0 → Operator → <Dial>
              9 → Repeat menu
```

## Webhook routes

Set `/answer` as the **Answer URL** in your Vobiz application settings.

| Method | Path | Description |
|---|---|---|
| POST | `/answer` | Main IVR menu |
| POST | `/ivr-main-choice` | Routes digit 1–4 / 0 / 9 to sub-menu |
| POST | `/ivr-sales` | Sales sub-menu |
| POST | `/ivr-sales-choice` | Routes sales digit choice |
| POST | `/ivr-support` | Technical support sub-menu |
| POST | `/ivr-support-choice` | Routes support digit choice |
| POST | `/ivr-support-ticket` | Raises a high-priority support ticket |
| POST | `/ivr-billing` | Billing sub-menu |
| POST | `/ivr-billing-choice` | Routes billing digit choice |
| POST | `/ivr-account` | Account management sub-menu |
| POST | `/ivr-account-choice` | Routes account digit choice |
| POST | `/ivr-account-cancel-confirm` | Double-confirms cancellation request |
| POST | `/ivr-operator` | Direct transfer to live operator |
| POST | `/dial-complete` | Callback after any `<Dial>` completes |
| POST | `/hangup` | Called when the call ends |

## Runtime config API

Update department transfer numbers without restarting the server.

| Method | Path | Description |
|---|---|---|
| GET | `/config` | View current department configuration |
| PUT | `/config/department/{dept}` | Update transfer number or enable/disable a department |
| GET | `/call-logs` | Full call history |
| GET | `/call-logs/analytics` | Which menu options are pressed most |

## Environment variables

| Variable | Required | Description |
|---|---|---|
| `VOBIZ_AUTH_ID` | Yes | Vobiz account Auth ID |
| `VOBIZ_AUTH_TOKEN` | Yes | Vobiz account Auth Token |
| `FROM_NUMBER` | Yes | Your Vobiz DID used as caller ID for transfers |
| `HTTP_PORT` | No | Server port (default: `8000`) |
| `PUBLIC_URL` | No | Production public URL — skips ngrok tunnel if set |
| `NGROK_AUTH_TOKEN` | No | ngrok auth token for local development tunneling |

<Tip>
For local development, leave `PUBLIC_URL` blank and set `NGROK_AUTH_TOKEN`. The server will start an ngrok tunnel automatically and print the public URL to the console.
</Tip>

## Key webhook patterns

The IVR is stateless. Each `<Redirect>` tells Vobiz to POST to the next route, and each handler returns a fresh XML response. Here is a representative pattern for the main menu:

```python
from flask import Flask, request, Response

app = Flask(__name__)

@app.route("/answer", methods=["POST"])
def answer():
    xml = """<?xml version="1.0" encoding="UTF-8"?>
<Response>
  <Gather action="/ivr-main-choice" method="POST" inputType="dtmf" timeout="5" numDigits="1">
    <Speak voice="WOMAN" language="en-US">
      Welcome. Press 1 for Sales, 2 for Technical Support,
      3 for Billing, 4 for Account Management, or 0 for the Operator.
    </Speak>
  </Gather>
  <Redirect>/answer</Redirect>
</Response>"""
    return Response(xml, mimetype="application/xml")


@app.route("/ivr-main-choice", methods=["POST"])
def ivr_main_choice():
    digit = request.form.get("Digits", "")
    destinations = {
        "1": "/ivr-sales",
        "2": "/ivr-support",
        "3": "/ivr-billing",
        "4": "/ivr-account",
        "0": "/ivr-operator",
        "9": "/answer",
    }
    redirect_to = destinations.get(digit, "/answer")
    xml = f"""<?xml version="1.0" encoding="UTF-8"?>
<Response>
  <Redirect>{redirect_to}</Redirect>
</Response>"""
    return Response(xml, mimetype="application/xml")
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Caller hears nothing after pressing a digit">
    Verify the `action` URL in your `<Gather>` is publicly reachable. If running locally, ensure your ngrok tunnel is active and `PUBLIC_URL` in `.env` points to the tunnel URL.
  </Accordion>
  <Accordion title="Transfer fails silently">
    Check that `FROM_NUMBER` in `.env` is a Vobiz DID you own and that it is set as the `callerId` in `<Dial>`. Vobiz rejects outbound `<Dial>` if the caller ID is not verified.
  </Accordion>
  <Accordion title="Calls not reaching my server">
    Go to the Vobiz Console → **Applications** and confirm the Answer URL is set to your public URL `/answer` with method `POST`. Inbound numbers must be linked to this application.
  </Accordion>
</AccordionGroup>
